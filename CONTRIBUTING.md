## How to Contribute

### Reporting Issues

If you encounter problems:
1. Check existing issues first
2. Create a new issue with:
   - Clear description of the problem
   - Steps to reproduce
   - Expected vs actual behavior
   - Your AWS region and account type (Free Plan vs Paid Plan)

### Suggesting Enhancements

Have ideas to improve the project?
1. Open an issue with the "enhancement" label
2. Describe your suggestion clearly
3. Explain why it would be useful

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test the CloudFormation template thoroughly
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

### Testing CloudFormation Changes

Before submitting:
- Test template deployment in a clean AWS account
- Verify all resources are created successfully
- Check that billing alarms work correctly
- Ensure cleanup (stack deletion) works properly

### Code Style

- Use clear, descriptive resource names
- Add comments for complex configurations
- Follow AWS CloudFormation best practices
- Keep templates simple and maintainable

## Questions?

Open an issue or reach out to the maintainers.
